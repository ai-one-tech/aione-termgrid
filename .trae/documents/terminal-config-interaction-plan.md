# 终端配置交互方案

## 现状分析

当前设置页面有两个 Tab：
- **网格布局**：设置行列、合并单元格
- **终端配置**：通过下拉框选择 cell，然后编辑该 cell 的标题、工作目录、命令、顺序、延迟、边框颜色

**问题**：
1. 终端配置 Tab 与网格布局 Tab 完全分离，用户无法直观知道当前编辑的是哪个格子
2. 合并后的单元格在终端配置下拉框中只是普通的 cell 列表项，没有视觉关联
3. 操作路径长：先记住格子编号 → 切换到终端配置 Tab → 在下拉框找到对应编号 → 编辑

---

## 目标

- 每个格子（含合并后的）都能直观地进行终端配置
- 操作简单、一次点击即可进入配置
- 视觉关联强，用户始终知道自己在配置哪个格子

---

## 推荐方案：网格可视化点击配置（方案一）

### 核心思路
**去掉"终端配置"Tab，改为在网格布局上直接点击格子进行配置。**

### 交互设计

#### 1. 布局预览区变为"可点击的配置入口"

```
┌─────────┬─────────┬─────────┬─────────┐
│    1    │    2    │    3    │    4    │  ← 点击任意格子打开配置面板
├─────────┴─────────┴─────────┘         │
│              5,6,7          │    8    │  ← 合并格点击后配置整个合并区域
├─────────┬─────────┬─────────┼─────────┤
│    9    │   10    │   11    │   12    │
├─────────┼─────────┼─────────┼─────────┤
│   13    │   14    │   15    │   16    │
└─────────┴─────────┴─────────┴─────────┘
```

#### 2. 点击后的交互 —— 右侧滑出配置抽屉

点击某个格子后，从右侧滑出一个配置抽屉（Sheet/Drawer），专门编辑该格子的终端配置：

```
┌─────────────────────────────┬─────────────────────┐
│                             │  ⚙️ 终端 1 配置      │
│                             ├─────────────────────┤
│                             │  标题               │
│      网格布局区域            │  [Terminal 1    ]   │
│      （保持可见，            │                     │
│       点击的格子高亮）        │  工作目录            │
│                             │  [.             ]   │
│                             │                     │
│                             │  命令               │
│                             │  [默认 ▼] [bash   ] │
│                             │                     │
│                             │  顺序 [1] 延迟 [0]  │
│                             │                     │
│                             │  边框颜色            │
│                             │  [🟢][🔵][🟡][🔴]  │
│                             │                     │
│                             │  [保存] [取消]      │
└─────────────────────────────┴─────────────────────┘
```

#### 3. 视觉反馈

- **点击的格子高亮**：边框变蓝或添加阴影，让用户明确知道在配置哪个格子
- **合并格特殊标识**：合并格上显示一个小图标（如 ⚙️ 或齿轮），提示可点击配置
- **已配置/未配置状态**：如果格子有自定义配置（非默认值），格子上显示一个小绿点或标题缩写

#### 4. 批量操作（可选增强）

- 支持 `Shift` 或 `Ctrl` 多选格子，批量设置相同的命令或工作目录

## 备选方案

### 方案二：网格布局 Tab 内嵌配置卡片

不滑出抽屉，而是在网格布局下方直接展开配置表单：

```
┌─────────┬─────────┐
│    1    │    2    │
├─────────┴─────────┤
│      3,4          │
└───────────────────┘

⚙️ 正在配置：格子 1
┌─────────────────────────────────────┐
│ 标题 [          ]  工作目录 [      ] │
│ 命令 [          ]  颜色 [🟢🔵🟡🔴]  │
│ [保存]                              │
└─────────────────────────────────────┘
```

**优点**：更紧凑，不需要额外的抽屉组件  
**缺点**：页面会跳动，配置复杂时表单可能很长

### 方案三：悬浮编辑按钮

每个格子上悬浮一个小的编辑按钮（hover 时显示），点击后弹出模态框：

```
┌─────────────────┐
│  1        [⚙️]  │  ← hover 显示齿轮按钮
├─────────────────┤
│  2        [⚙️]  │
└─────────────────┘
```

**优点**：不占用空间，界面干净  
**缺点**：需要 hover 才能发现功能，移动端不友好

---

## 推荐实现：方案一（右侧抽屉）

理由：
1. **空间充足**：抽屉可以放下完整的配置表单，不拥挤
2. **上下文保持**：左侧网格始终可见，用户不会迷失
3. **操作直观**：点哪里配哪里，零学习成本
4. **扩展性好**：后续可以加更多配置项（环境变量、启动参数等）

---

## 实现要点

### 数据结构

当前 `cells` 数组是按顺序存储的，合并格子的配置应该存储在 `cells` 中对应的 cell 上。合并后，被合并的 cell 的配置可以忽略（或者合并时自动同步）。

需要确认：
- 合并格子后，`cells` 数组如何对应？当前代码中 `cells` 似乎还是保持原始数量，只是视觉上合并了。
- **建议**：合并后，被合并的 cell 可以标记为 `hidden: true`，配置时只显示顶层 cell。

### 状态管理

```typescript
const [editingCellId, setEditingCellId] = useState<string | null>(null);
const [showConfigDrawer, setShowConfigDrawer] = useState(false);

const handleCellClick = (cellId: string) => {
  setEditingCellId(cellId);
  setShowConfigDrawer(true);
};
```

### 组件拆分

- `SettingsPanel`：主面板，包含网格布局和合并功能
- `CellConfigDrawer`：右侧抽屉，专门负责单个 cell 的配置编辑

### 交互细节

1. **单击格子**：打开配置抽屉（如果当前不是在选择合并模式）
2. **合并选择模式**：需要一个明确的切换按钮进入"合并选择模式"，避免和点击配置冲突
3. **抽屉关闭**：自动保存或提示保存（根据产品习惯）

## 总结

| 方案 | 操作复杂度 | 直观程度 | 空间利用 | 推荐度 |
|------|-----------|---------|---------|--------|
| 方案一：右侧抽屉 | 低 | 高 | 高 | ⭐⭐⭐ 强烈推荐 |
| 方案二：下方展开 | 低 | 中 | 中 | ⭐⭐ |
| 方案三：悬浮按钮 | 中 | 中 | 高 | ⭐⭐ |

**最终推荐**：采用方案一，将终端配置整合到网格布局的视觉交互中，点击格子即打开右侧配置抽屉，实现"所见即所配"的体验。
